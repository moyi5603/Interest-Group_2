## 一、产品概述

### 1.1 背景

企业员工有跨部门兴趣社交与活动参与需求。工作台已有「谋发展 → 兴趣小组」入口及员工档案，但尚未形成完整的**发现小组 → 参与活动 → 自发建组**闭环。

本产品为**已登录员工**提供：维护个人兴趣、发现推荐小组与活动、报名参与、创建自发小组并完成合规报备；组长可发布与管理活动；系统通过规则推荐与轻量 AI 降低组织成本。

### 1.2 产品定位

- **形态**：嵌入 EXP 智能体的**移动端优先**能力模块，交互风格与悦文化等活动产品保持一致。
- **用户**：全公司登录员工；官方精品小组由运营在后台维护，员工端不提供「创建官方组」入口。
- **阶段**：当前为 **MVP**，用于需求对齐与体验验证；部分能力为规则/mock，正式环境需对接企业服务。

### 1.3 核心价值

**对员工**：按兴趣发现内容与活动，可先报名、后加入，降低参与门槛。  
**对组长**：快速建组、发活动，AI 辅助撰写活动文案。  
**对企业**：自发组「先上线、后报备」，在合规前提下激发基层活力。

## 二、目标与范围

### 2.1 MVP 要达成的目标

1. 员工能维护兴趣标签，并看到**带理由说明**的小组与活动推荐。  
2. 同时支持**官方精品小组**与**员工自发小组**；自发组创建后立即可被发现，并在规定期限内提醒完成工会/HR 报备。  
3. 活动支持四种时间形态：**单次、长期开放、固定周期、系列多场次**。  
4. **报名与加入小组相互独立**：未加入小组也可以报名该组下的活动。  
5. 提供三类 AI 辅助：**智能推荐、活动文案生成、对话查询**（不含搭子/组队匹配）。

### 2.2 建议观测的成功指标

- 至少选择 2 个兴趣标签的员工占比（标签完善率）。  
- 浏览推荐后完成「加入小组」的转化率。  
- 活动详情页报名成功率。  
- 自发组创建后 7 日内完成「我已报备」的比例。  
- 组长人均发布活动次数。

### 2.3 明确不做（MVP）

- 搭子匹配、组队推荐。  
- 在兴趣小组模块内发放积分卡（积分由荣誉体系按赛事、表彰等场景单独配置）。  
- 真实 HR 对接、IM 消息推送、签到、日历同步、运营审阅后台、向量语义检索、全量大模型对话。  

以上能力列入二期规划。

## 三、用户角色

**普通员工**  
维护个人兴趣标签；浏览、搜索、加入小组；**不加入小组也可报名活动**；查看与管理自己的报名记录；创建员工自发小组。

**小组创建人（组长）**  
一般为自发组的创建人，或官方组指定的负责人。可编辑小组资料、发布与终止活动、解散小组；自发组需关注报备提醒。

**官方运营**  
维护官方精品小组与推荐位；事后审阅自发组报备情况（MVP 阶段员工侧仅做报备确认，无完整运营后台）。

## 四、典型用户场景

**发现与标签**  
员工在「我的兴趣」中选择或取消标签；标签较少时系统给出建议标签，可一键添加或忽略。首页展示推荐小组及匹配理由，支持换一批。标签在「我的兴趣」与员工档案中保持一致；**他人查看员工主页时不展示兴趣标签**，仅展示其已加入的小组名称。

**小组**  
员工通过首页推荐、发现页搜索加入公开小组；满员或邀请制小组给出明确提示。任意员工可创建自发组，填写名称、简介、封面与标签后立即上线。组长在详情页看到报备提醒，确认已向工会/HR 报备后提醒消失。组长可编辑资料、解散小组；已加入且非组长的成员可**退出小组**，退出后其在**该小组下所有活动的报名一并取消**。

**活动**  
员工在活动详情查看时间、地点、说明与场次信息并完成报名；周期类、系列类活动可按规则选择多个场次（详见第六节）。已报名者可继续追加场次或取消报名（需二次确认）。组长在单页流程中选择活动类型、填写信息，并可让 AI 一次生成多条标题/描述备选后填入。组长对进行中的周期/系列活动可执行「终止」，终止后不再接受新报名，未举办场次取消，已有报名作废。

**AI 与汇总**  
员工可用自然语言询问「推荐跑步小组」「下周有什么活动」「我加入了哪些组」等，系统返回小组或活动卡片。首页展示「我的小组」摘要与「近期活动」预览；「我的报名」集中展示参与记录。

## 五、功能需求说明

### 5.1 兴趣标签

员工在「我的兴趣」页按分类浏览并勾选标签，随时增删。当有效标签少于 2 个时，在页面展示系统建议的标签，员工可选择添加或永久忽略某条建议。标签变更后，首页推荐的小组列表应随之变化。

员工在档案侧维护的兴趣标签，与「我的兴趣」页保持同步，避免两处信息不一致。除本人与推荐服务外，其他员工无法在同事主页看到其兴趣标签列表。

### 5.2 小组发现与成员关系

**首页推荐**  
根据员工已选标签、所在部门、小组类型（如官方精品组）等综合打分，排出未加入的候选小组，并展示可读推荐理由（例如与哪些标签匹配）。已加入的小组不再出现在推荐中。支持「换一批」刷新列表。

**发现与加入**  
提供搜索与列表浏览。公开小组可直接申请加入；仅本部门可见的小组对非本部门员工不可见；邀请制小组不可主动加入，需提示联系组长。单小组成员上限为 **100 人**，满员后不可再加入。

**小组详情**  
展示封面、简介、兴趣标签、成员规模，以及按类型筛选的活动列表（全部 / 单次 / 周期 / 系列）。非成员且小组未满、可加入时，展示「加入小组」。已加入且非组长的成员，展示「退出小组」，操作前需二次确认。组长展示「发布活动」入口；组长在页面底部还可「编辑小组」「解散小组」，解散需确认，解散后小组下线并联动终止组内进行中的活动。

**创建自发组**  
登录员工均可创建，无需额外审批门槛。创建人自动成为组长并计入成员。创建后小组立即对外可见（当前版本创建页不提供可见范围选项，默认为全公司公开）。创建成功后进入该小组详情页。

**自发组报备**  
创建完成后，小组详情顶部展示报备提醒，说明须在创建后 **7 日内**向工会/HR 完成报备。组长点击「我已报备」后，提醒条消失。未报备不自动下架小组（除非二期运营介入）。

### 5.3 活动

**谁能发布**  
仅小组创建人（组长）可新建、编辑、终止活动；普通成员与非小组成员均看不到发布入口。若通过链接误入发布页，应提示无权限并引导返回。

**活动类型**  

- **单次活动**：有明确的开始、结束时间与报名截止时间；受总人数上限约束。  
- **长期开放活动**：长期接受报名，无固定场次结束概念。  
- **固定周期活动**：按每周/每月等规则重复举办，员工按「场次」报名。  
- **系列活动**：多场次构成一条活动线，发布时由组长选择报名方式（见下）。

**报名与加入的关系**  
只要员工能看见该活动，且活动仍在发布状态、未报满、未过报名截止或场次规则允许，即可点击报名，**无需先加入所属小组**。加入小组仍可作为可选动作，便于持续关注该组动态。

**单次与长期开放**  
在详情页一键报名；满员或已过截止时，按钮不可用并说明原因。

**系列活动报名方式**  
组长发布时可二选一：  

1. **整场报名**：适合赛程连贯的活动（如篮球赛初赛到决赛）。员工只需报名一次即视为参与全部场次；仅在该系列**第一场开始前**可新报名，首场开始后不再接受新报名。  
2. **按场次报名**：适合各场相对独立的活动（如系列分享）。员工可只报其中若干场。

**周期活动与系列（按场次）**  
报名时可勾选多个未来场次。以当前日期起 **三个月内**、尚未结束且未满员的场次可选；三个月之后的场次在列表中展示但不可选，并说明「仅开放近三个月内报名」。已报场次、已满场次分区展示；支持在已报基础上「继续报名」更多场次。取消报名需二次确认。

**活动详情展示**  
包含标题、封面、时间地点、说明、互动数据（赞/藏/评等展示位）、报名状态与场次列表。已被组长终止的活动，明确提示不再接受报名，未举办场次视为取消。

**我的报名与首页活动**  
「我的报名」按员工维度汇总其已报场次与活动，可跳转回活动详情。首页「近期活动」以**活动**为卡片单位，同一周期/系列活动不因多个场次重复出现多张卡；不展示已全部结束的活动；周期类展示规则摘要（如「每周三 19:00」），系列类展示场次数与下一场时间。

### 5.4 AI 能力

**智能推荐**  
基于标签匹配、部门、小组类型等规则排序，并生成简短推荐理由文案；不依赖向量检索。

**活动文案生成**  
组长在发布/编辑活动时，可请求 AI 一次生成多条标题与描述备选，点击即可填入表单；生成失败时提示，并可手工填写。

**对话助手**  
在兴趣小组对话页，员工用自然语言提问。MVP 支持：推荐某类小组、查询近期可报名活动、查看我已加入的小组、如何发起活动等常见意图，以卡片列表形式返回结果。

### 5.5 导航与入口

**模块内主要页面**  
兴趣小组首页、发现/加入小组、创建小组、小组详情、编辑小组、发布/编辑活动、活动详情、我的报名、我的兴趣标签、AI 对话、各类列表页（我的小组全集、近期活动全集等）。

**模块外入口**  
智能体广场、悦文化等相关入口；首页推荐问题；员工档案/员工详情中的「已加入小组」列表，点击可进入对应小组详情。

### 5.6 反馈与体验

操作成功、失败或需提示时，使用**屏幕居中、纯文字**的轻提示，不遮挡核心内容。退出小组、取消报名、解散小组等不可逆操作须**二次确认**。全模块字号与信息层级与首页保持一致，保证移动端可读性。整体视觉与悦文化活动产品一致：卡片化列表、圆角主按钮、组长底部操作栏等。

## 六、业务规则（文字说明）

### 6.1 加入、退出与报名

员工可以把「加入小组」和「报名活动」当作两件独立的事：想参加某场活动，不必先点加入；想长期关注某个小组，可以再选择加入。

退出小组时，系统应自动取消其在该小组下所有活动的报名，避免人已退出却仍占用名额。

邀请制小组不允许自行加入，需联系组长。小组成员达到 100 人上限后，不再接受新成员。

### 6.2 权限分工

创建自发小组：任意登录员工。  
创建官方精品小组：仅运营侧，员工端无入口。  

发布、编辑、终止活动，以及编辑小组资料、解散小组：仅该小组创建人（组长）。  
普通成员：可浏览、报名，不可发布活动。  
非小组成员：在可见范围内可浏览活动并报名，不可发布活动。

### 6.3 可见性

小组可见范围分为：**全公司公开**、**仅指定部门**、**仅邀请**。非本部门员工不应看到「仅部门」的小组；邀请制小组不在公开搜索中展示。

活动默认随小组可见；员工能看见活动详情，即按前述报名规则判断是否可报。

### 6.4 推荐逻辑

推荐小组时，优先匹配员工已选兴趣标签；标签重合越多，排序越靠前。若小组限定部门，同部门员工可获得额外权重。官方精品组可适当优先展示。已经加入的小组从推荐列表中排除，并给出人类可读的推荐理由（如「与你关注的 #跑步 匹配」）。

推荐活动时，优先展示员工已加入小组里即将开始的场次；其次展示未加入小组、但标签匹配且公开的活动；长期开放类活动可单独区块展示。

### 6.5 积分与荣誉

兴趣小组模块**不负责发放积分**。积分、勋章等激励由公司**荣誉体系**按赛事结果、表彰规则等统一配置（例如篮球赛冠军队伍成员获得固定积分）。未来可考虑把某次活动或小组作为荣誉规则的触发来源，但**不默认每场活动都绑积分**。

### 6.6 异常与边界

| 情况 | 产品行为 |
|------|----------|
| 活动名额已满 | 报名不可用，提示已满 |
| 已过报名截止或首场已开始（整场系列） | 报名不可用，说明原因 |
| 某场次已取消 | 列表中灰显，不可选 |
| 邀请制小组、无邀请 | 提示联系组长 |
| 员工无任何兴趣标签 | 推荐降级为热门官方组，并引导完善标签 |
| AI 文案生成失败 | 提示失败，允许手工输入 |

## 八、交互与视觉原则

- 移动端优先，单列阅读，顶部返回与页面标题。  
- 列表以卡片呈现；活动卡片含封面、标题、状态、时间地点、报名态。  
- 主操作使用醒目的主按钮；破坏性操作（退出、取消报名、解散）使用警示样式并二次确认。  
- 操作反馈使用居中文字提示。  
- 详细组件与版式参考《悦文化 APP UI 风格规范》。

## 九、非功能需求

- **权限与安全**：仅登录员工可访问；兴趣标签不对其他同事公开展示。  
- **性能**：正式环境下列表首屏加载目标 2 秒内（MVP 原型不作硬性要求）。  
- **兼容性**：主流移动浏览器。

## 十、验收标准（MVP）

**功能**  
- 增删标签后，推荐小组及理由随之合理变化。  
- 四种活动类型在发布页字段互不冲突，周期类能展示足够未来场次。  
- 未加入小组的用户可对可见活动完成报名。  
- 成员退出小组后，该组活动报名全部取消。  
- 组长可发布活动、终止周期/系列活动、解散小组；非组长无发布入口。  
- 系列「整场 / 按场次」、近三个月场次选择规则符合上文描述。  
- 自发组创建后可被推荐/发现；报备提醒与「我已报备」交互正确。  
- 部门限定小组对非本部门员工不可见。  
- 对话助手覆盖推荐小组、查活动、我的小组等主路径。  
- 他人主页不展示兴趣标签。  

**体验**  
- 关键操作有明确文字反馈。  
- 退出、取消报名、解散有二次确认。  
- 各页字号与信息层级统一、易读。

## 十一、版本规划

### MVP（当前）

兴趣标签、小组推荐与加入、自发组创建与报备提醒、退出小组、四种活动类型、报名与加入解耦、多场次报名与取消、组长发布与终止活动、AI 推荐/文案/对话（规则级）、我的报名与首页聚合展示。

### 二期（方向）

- 对接 IM：报名成功、活动开始前提醒、报备临期、场次变更等消息与跳转。  
- 运营后台：自发组报备审阅、推荐位配置、异常小组处理。  
- 权限扩展：组长转让、管理员代发、建组审批与白名单。  
- 智能化：语义推荐、真实大模型、搭子匹配。  
- 参与深化：签到、企业日历同步、节假日顺延。  
- 与荣誉体系联动：按规则触发积分/勋章，仍避免「每场活动默认发分」。  
- 与企业 HR、主数据、持久化服务对接。

## 十二、名词解释

| 名词 | 含义 |
|------|------|
| 官方精品小组 | 由公司运营创建和维护的标杆小组 |
| 自发小组 | 由员工创建的兴趣小组，创建人即首任组长 |
| 场次 | 周期或系列活动中的单次会议/单场实例 |
| 整场报名 | 系列活动中报一次名即参与全部场次的方式 |
| 按场次报名 | 系列或周期活动中，员工自由选择其中若干场参与 |
| 荣誉体系 | 公司统一的积分、勋章、表彰配置与发放能力 |
