## 1. 产品定位与入口

| 项 | 说明 |
|----|------|
| 终端 | **悦享福利 App**（移动端） |
| 入口 | **智能体 → 悦文化**（非独立 App 图标为主入口） |
| 定位 | 企业文化数字化：资讯 + 活动 + 勋章 + AI 助手 + IM 通知 |
| 后台依赖 | 活动/勋章由 **EXP 活动管理 + 勋章发放** 后台配置，C 端只消费数据 |

与当前本仓库 `humanistic-care` 原型的关系：原型模拟「EXP 智能体」壳层；悦文化是智能体下的**子业务 Tab/模块**，活动卡片/对话模式应与此文档一致。

## 2. 整体视觉与信息架构原则

### 2.1 设计语言（从需求归纳）

- **卡片化**：活动、资讯、AI 回复均以**结构化卡片**呈现，少用大段纯列表文字。
- **与 EXP 文章体系一致**：资讯列表/详情、互动数据、装修组件逻辑**复用既有文章系统 UI**；活动是文章能力平行扩展。
- **移动优先、碎片化浏览**：首页聚合入口 + 少量预览（2 条）+「查看全部」深链列表。
- **底部常驻 AI**：首页底部「试试这样问」+ 输入栏；深页可跳转完整对话页。
- **荣誉可视化**：勋章出现在活动卡片、我的报名、我的勋章网格；增强仪式感。

### 2.2 顶层 IA（悦文化模块）

```
底部 Tab：悦文化（首页）
├── 快捷入口 ×4：活动中心 | 企业资讯 | 我的报名 | 我的勋章
├── 最新活动预览（≤2，未结束）→ 查看全部 → 活动中心
├── 最新资讯预览（≤2）→ 查看全部 → 企业资讯
└── 底部 AI 区：话题推荐(3) + 输入栏 → AI 对话页

并行能力（非 Tab 内页）：IM 沟通引擎推送（报名/勋章等）
```

## 3. C 端列表布局模式（EXP 装修 / 活动组件）

活动（及类似内容）在 SaaS/APP 装修支持多种**列表排布**，与文章组件同一套配置思路：

| 模式 | 说明 |
|------|------|
| **大图模式** | 封面突出，适合重点运营活动 |
| **一行一个** | 单列卡片列表（**资讯默认**；悦文化文章明确「同 exp 一行一列」） |
| **一行两个** | 双列网格（**我的勋章**用双列展示勋章） |
| **横向滑动** | 横滑卡片带，适合首页推荐位 |

**总首页**：按**产品线归类**分类（与文章总首页一致）。

**互动**：点赞 / 收藏 / 评论支持**按应用（产品线）切换**统计与展示。

## 4. 页面级风格说明

### 4.1 首页（悦文化）— P0

| 区域 | 规范 |
|------|------|
| 位置 | 底部 Tab「悦文化」 |
| 快捷入口 | **4 宫格**：活动中心、企业资讯、我的报名、我的勋章 |
| 最新活动 | 最多 **2 条**未结束；字段与活动中心列表一致；**查看全部** → 活动中心 |
| 最新资讯 | 最多 **2 条**；字段与企业资讯列表一致；**查看全部** → 企业资讯 |
| AI 区 | **「试试这样问」**：每次 **3 条**话题，可刷新；点击直发 AI 页 |
| 输入栏 | 文字输入，发送 → **跳转 AI 智能体对话页** |

### 4.2 活动中心 — P0

| 元素 | 规范 |
|------|------|
| 筛选 Tab | **全部 / 未开始 / 进行中 / 已结束**（不按产品线拆，全量可见活动） |
| 排序 | 创建时间倒序 |
| 可见性 | 仅展示当前用户可见范围（后台配置） |

**活动卡片字段（自上而下）**：

1. **封面图** — 已结束活动叠加**遮罩**提示  
2. **标题** — 活动名称  
3. **状态标签** — 未开始 / 进行中 / 已结束  
4. **时间行** — 开始时间、结束时间（两行）  
5. **地点行**  
6. **底部互动区** — 点赞数 / 收藏数 / 评论数  
7. **报名按钮** — 进入两步报名流程  

**勋章叠加**：若活动中获得勋章，展示在卡片上；多个勋章**横向并排**。

点击卡片 → **活动详情页**。

### 4.3 活动详情 & 报名 — P0

**信息展示顺序**（固定）：

1. 活动时间（开始–结束）  
2. 报名时间（创建时间起–截止）  
3. 地点  
4. 名额（多报名类型为各类型之和）  
5. 发起人（姓名 + 末级部门）  
6. 联系电话（**可点击拨号**）  
7. 活动详情（富文本描述）  

**报名**：**两步弹窗**

- Step 1：选择报名类型及剩余名额（仅一种类型时**跳过**）  
- Step 2：确认信息并提交  

**底部互动栏**：

- 点赞 / 收藏 / 评论数  
- **报名按钮状态**：未报名且期内 →「立即报名」；已报名 → 显示活动状态（未开始/进行中/已结束）**禁用态「已报名」**  

### 4.4 企业资讯 — P0

- 分类：**全部** + 文章分类（按产品线归堆，同 EXP 文章首页）  
- 列表：**一行一列**卡片（标题 / 时间 / 阅读量等，同 EXP）  
- 点击 → 资讯详情  

### 4.5 资讯详情 — P0

- **基线**：同 EXP 文章详情（发布时间、正文、评论、底部互动栏）  
- **增量**：顶部 **AI 提炼摘要** 折叠区 — 展开后流式输出：一句话主题、关键词、概要  

底部互动：点赞 / 收藏 / 评论（与文章一致）。

### 4.6 我的报名 — P0/P1

- 列表：封面、标题、状态；驳回显示**原因**，可再次报名  
- 筛选：按**审核状态**（默认全部）  
- **取消报名**：待审核可取消；开始前 **24 小时**内可取消；二次确认 → 状态「已取消」  
- 卡片可展示**已获勋章**（多个并排）  
- 点击 → 活动详情  

### 4.7 我的勋章 — P0/P1

- 布局：**双列网格**  
- 单元格：勋章图、名称、获得日期；按获得日期倒序  
- 点击 → **详情弹窗**：名称、描述、获得时间、关联活动名称  

### 4.8 AI 悦文化智能体对话 — P0

**入口**：首页输入发送 / 点击推荐话题。

**消息类型（回复体）**：

| 类型 | 用途 |
|------|------|
| `text` | 纯文字（指引、确认、兜底欢迎语） |
| `activity_list` | 活动卡片列表（可进详情/报名） |
| `article_list` | 资讯卡片列表 |
| `cancel_list` | 已报名列表 + 卡片上「取消报名」 |
| `summary_list` | 资讯列表 +「一键总结」按钮 |

**输入区**：文字发送；**语音按钮**（v40 占位，录音浮层）。

**意图优先级**：指令式（点赞/收藏/评论）> 精确话题 > 关键词分类 A–E > 兜底 F。

**话题推荐（首页）**：6 组 × 3 条 = 18 条，对应活动查询、资讯查询、报名记录、取消报名、内容总结、操作指引。

**卡片数量上限（对话内）**：

- 活动列表：最多 **3** 条未结束  
- 资讯列表：最多 **4** 条  

**兜底文案**：  
「您好！我是悦文化小助手，可以帮您查询企业活动、资讯和相关信息。请问有什么可以帮您？」

### 4.9 IM 沟通引擎 — P0

- 会话列表：头像、名称、摘要、时间、未读（支持私聊/群聊）  
- 推送节点：报名成功 / 待处理 / 已取消 / **勋章颁发**  
- 推送内容：活动名、状态、时间、说明；可点击进入详情或我的勋章

## 5. B 端（EXP 后台）风格要点 — 供对照

> C 端原型虽不实现后台，但需知「数据从哪来」。

- **布局**：顶栏「概览 / 文章 / **活动**」+ 左侧菜单；主区**左分类树 + 右列表**（与文章一致）。  
- **列表**：筛选区 + 排序 Tab（最新 / 点赞 / 评论）+ 表格 + 行内「⋯」菜单。  
- **编辑**：弹窗两大块 — **活动信息** + **报名信息**；富文本详情；多报名类型可增删行。  
- **勋章**：卡片列表选勋章 / 新建勋章（图标上传+预览）；发放方式手动；发放记录可查。  
- **装修**：APP/SaaS 增加「活动推荐」组件，配置方式**借鉴文章组件**。

## 6. 组件级交互惯例（实现时复用）

| 模式 | 行为 |
|------|------|
| 卡片点击 | 进详情（活动/文章） |
| 主按钮 | 报名 / 立即报名；成功后变灰「已报名」 |
|  destructive | 取消报名 → **Alert 二次确认** |
| 弹窗 | 报名两步、勋章详情、发放详情 |
| 折叠 | AI 摘要、AI 提炼下拉 |
| 空态 | 列表无数据文案（如报名人员「暂无匹配人员」） |
| 状态标签 | 未开始 / 进行中 / 已结束；审核态单独筛 |

## 7. 与兴趣小组设计的对齐建议

兴趣小组若挂在 **EXP 智能体 / 悦文化** 体系内，建议：

1. **列表**：活动型内容用「一行一个」活动卡片；小组列表可用同结构（封面/标题/标签/成员数/底部互动）。  
2. **首页**：4 宫格旁可增加第 5 入口「兴趣小组」，或并入「活动中心」Tab 子分类。  
3. **AI**：复用「试试这样问」3 条 + `activity_list` 卡片回复；对话页推荐小组用 `activity_list` 变体或新 `group_list` 类型。  
4. **勋章**：小组活动完成发放勋章时，遵循**卡片上并排展示**规则。  
5. **可见性**：与活动「可见范围」一致 — public / 部门 / 邀请制映射到列表过滤。
