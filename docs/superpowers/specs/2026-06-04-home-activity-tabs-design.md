# 首页活动 Tab 模块设计

日期：2026-06-04
载体：`site/` 移动员工端原型（`site/assets/9e8f0b88-…js` 的 `HomeTab`）

## 背景

移动首页当前把活动按「为你推荐」（横滑）与「本周活动」（纵向大卡）两个独立区块呈现，缺少「热门活动」维度，区块分散、纵向占用高。需要将三类活动收敛到一个 Tab 切换模块，提升信息密度与可比性。

## 目标

在首页中部用一个分段控件（Segmented）+ 纵向活动大卡列表，承载「推荐 / 最新 / 热门」三类活动，替换原「为你推荐」「本周活动」两个区块。

## 方案（方案 A）

只整合「为你推荐 + 本周活动」两块为 Tab 模块；「热门小组」「往期精彩回顾」保持不变。

### 位置

我的活动/我的小组快捷入口 之下，热门小组 之上。

### 结构

- `SectionHeader`：标题「活动」，右侧「全部 →」跳转 `allActs`。
- 胶囊分段控件：`推荐 | 最新 | 热门`，选中项填充 `--ink`、白字，未选 `--ink-2`；切换有高亮过渡。
- 下方纵向 `ActivityCard` 大卡列表，随当前 Tab 切换数据，页面不跳转。当前 Tab 由 `useState` 维护，默认「推荐」。

### 三 Tab 数据规则

| Tab | 内容 | 排序 | 数量 |
|-----|------|------|------|
| 推荐（默认） | 人工挑选 `recs`（含推荐理由） | 沿用数组顺序 | 3 |
| 最新 | `status === 'upcoming'` | 日期升序（最近开始在前） | 3 |
| 热门 | `status === 'upcoming'` | `likes` 降序，并列再按 `signed` 降序 | 3 |

日期解析复用 `MM月DD日` 解析逻辑（与 `AllActivities` 一致，mock today = 2026-06-03）。

### 推荐理由保留

纵向列表不再使用横滑 `RecCard`。推荐 Tab 中，`ActivityCard` 通过 `recReason` 将 AI 理由叠加在封面图左下角（标题上方）的半透明小药丸（`Sparkles` + 理由，`clamp1`），不增加卡片高度、不影响正文日期/地点/报名进度。最新/热门 Tab 不传 `recReason`，无理由。

### 空态

任一 Tab 无数据时渲染 `Empty`，文案分别为「暂无推荐活动 / 暂无最新活动 / 暂无热门活动」。

## 不在范围内

- 不改「热门小组」「往期精彩回顾」。
- 不新增后端/接口；纯原型 mock。
- 不改 `AllActivities` 全部活动页。

## 验收

- 首页中部出现单一活动模块，默认显示推荐 3 条（带理由条）。
- 点「最新」「热门」即时切换列表，无页面跳转；热门按点赞排序。
- 「全部 →」进入全部活动页。
- 原「为你推荐」「本周活动」两个独立区块不再出现。
