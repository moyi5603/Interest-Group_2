# 报名前置：必须加入小组才能报名（含审核联动）

日期：2026-06-04
范围：C 端员工报名流程 + 与 PC 管理端审核联动。仅 `site/` 原型。

## 背景与目标

现有业务原则「报名 ≠ 加入：可直接报名」被**反转**为新规则：

> **必须先加入活动所属小组，才能报名该小组的活动。**

需覆盖小组的两种加入类型：`free`（自由加入）与 `approve`（审核加入）。审核加入采用**硬门槛**：未通过审核不能报名。审核流程与 PC 管理端的入组审核队列**联动**。

> 注意：本 spec 取代 `exp-interest-group-ux` skill 中「报名 ≠ 加入」原则。实现后需同步更新该 skill。

## 用户与小组状态 → 报名按钮矩阵

报名两处入口（活动卡片快捷按钮、活动详情底部 CTA）逻辑一致：

| 用户 / 小组状态 | 按钮文案 | 点击行为 |
|---|---|---|
| 已入组 | `立即报名` / `选场次报名` | 直接报名（现状不变） |
| 未入组 · 自由加入 | `报名并加入小组` | 一键：入组 + 报名，toast「已加入小组，报名成功」 |
| 未入组 · 审核加入 · 未申请 | `报名并加入小组` | 仅提交入组申请（**不报名**）；toast +常驻提示「已提交申请，等待小组审核，通过后可报名」 |
| 未入组 · 审核加入 · 待审核 | `审核中…`（置灰禁用） | 不可点；下方常驻提示同上 |
| 审核通过后（已入组） | `立即报名` | 用户再点一次正式报名 |

- 卡片快捷按钮文案可精简：自由加入未入组 → `报名+入组`；审核加入未申请 → `报名+入组`；待审核 → `审核中`（禁用）。点击行为与详情页一致。
- 小组详情加入按钮同步三态：`加入小组`/`申请加入` → `审核中`（禁用）→ `已加入`。

## 审核联动机制（决策1）

架构事实：`Showcase` 中移动端（`Phone`）与 PC（`AdminApp`）**互斥渲染**，切 Tab 卸载另一端并各自从 `window.DB` 重新初始化。两端不同时存在。

因此以 `window.DB` 为共享数据源实现联动：

1. C 端点「报名并加入小组」(审核组) → 向 `DB.joinRequests` 推一条本人申请
   `{ id, gid, name: DB.ME.name, dept, appliedAt: '刚刚', status: 'pending', self: true }`，
   并把该小组对本人置 `pending = true`（写入 `DB.groups` 与本地 state）。
2. 切到 PC → `AdminApp` 从 `DB.joinRequests` 初始化，新申请出现在「待审核」队列。
3. PC 点「通过」`approveJoin` → 写回 `DB`：该 `self` 申请对应小组 `joined = true, pending = false`，`members + 1`，移除申请。
   点「拒绝」`rejectJoin` → 该小组 `pending = false`（回未入组），移除申请。
   「全部通过」`approveAllJoin` → 同「通过」逐条写回。
4. 切回 C 端 → 从 `DB.groups` 重新初始化，小组已「已加入」，报名按钮变 `立即报名`。

**原型约束（如实告知用户）**：联动非实时，而是「切到 PC 审批 → 切回 C 端生效」。纯 C 端不切 PC 时停留在「审核中」，符合「等待审核」语义。

## 退出小组连带取消报名（决策2）

C 端小组详情点「已加入」退出时：

- 若该组有本人已报名活动（含场次报名）→ 二次确认弹窗：
  「退出后将取消你在该小组 N 个活动的报名，确认退出？」→ 确认则退出 + 取消这些报名（`joinedByMe=false`，`signed--`；场次清空）。
- 无报名 → 直接退出。

## 数据模型改动

- `DB.groups[i]` 增字段 `pending: boolean`（本人待审核态；默认 false）。`joined` 仍表示本人是否已是成员。
- `DB.joinRequests[i]` 本人申请条目增 `self: true` 标记。
- 修复现有 bug：C 端 `toggleJoin` 对审核组点「申请加入」时不应立即 `joined=true`，应进入 `pending`。

三态判定（本人对某小组）：
- 未入组：`!joined && !pending`
- 待审核：`pending`
- 已入组：`joined`

## 边界与非目标

- C 端报名本身仍为会话内本地态（切 Tab 重置），与现状一致；本次仅让**成员资格 + 入组申请**跨 Tab 经 `DB` 持久联动。
- 不引入真实后端 / 计时器自动审批。
- 不改推荐、积分、自发组等其他模块。

## 验收

1. 自由加入小组：活动详情/卡片显示「报名并加入小组」，点击后入组 + 报名成功，按钮变「取消报名」。
2. 审核加入小组：点击「报名并加入小组」→ 未报名，进入「审核中」禁用态 + 提示；切 PC「待审核」可见本人申请；通过后切回 C 端按钮变「立即报名」；再点报名成功。
3. 拒绝：切回 C 端恢复「报名并加入小组」。
4. 退出已加入小组且有报名 → 二次确认并连带取消报名。
5. 无 lint 报错。
